# 女媧工坊 · Nuwa Studio

從一堆文件裡,煉出一個能對話的人。

本地端的人物思維蒸餾與對話工具,方法論改編自 [nuwa-skill(女媧 · Skill造人術)](https://github.com/alchaincyf/nuwa-skill)。把想蒸餾對象的相關文件丟進資料夾,啟動蒸餾,完成後即可與該人物對話、設定情境條件對話,或預測此人在特定情境下的反應。

## 安裝

需要 **Node.js 20 以上**。

```bash
git clone https://github.com/chung223/mindkiln.git
cd mindkiln
npm install
npm start
# 開啟 http://localhost:5723
```

首次使用:點左下角「設定」填入 Anthropic 或 MiniMax API 金鑰(或事先 `export ANTHROPIC_API_KEY=...`)。金鑰只存在你本機的資料夾,不會外流。

### 用 Docker(免裝 Node)

```bash
docker build -t mindkiln .
docker run -p 5723:5723 -v ~/nuwa-data:/data mindkiln
# 開啟 http://localhost:5723
```

資料掛在 `~/nuwa-data`(容器內 `/data`),重建映像也不會遺失。

### 環境變數(選填,可寫進專案根目錄的 `.env`,會自動載入)

| 變數 | 說明 | 預設 |
|------|------|------|
| `NUWA_DATA_DIR` | 資料存放位置。建議指到 repo 外(例:`~/nuwa-data`),避免混進版控,也方便備份 / 雲端同步。 | 專案內 `./data` |
| `PORT` | 伺服器埠 | `5723` |
| `HOST` | 綁定位址。預設只綁 `127.0.0.1`(本機安全);容器或需對外時設 `0.0.0.0`。 | `127.0.0.1` |
| `ANTHROPIC_API_KEY` | Anthropic 金鑰(也可在介面設定填) | — |
| `NUWA_PASSWORD` | 登入密碼。設了就啟用密碼保護,優先於介面設定且不落地。**對外部署務必設定。** | — |

複製 `.env.example` 成 `.env` 即可。`.env` 已被 `.gitignore` 排除。

### 部署到公開網路

預設沒有帳號密碼——只適合本機。要放上公網前,**務必先加登入保護**,二擇一:

- **環境變數(建議雲端 / 容器用)**:設定 `NUWA_PASSWORD=你的長密碼`,所有人進站都得先輸入它。
- **介面設定**:左下角「設定 → 登入保護」勾選「需要密碼才能使用」並設定密碼。

密碼以 scrypt 雜湊存於 `data/config.json`(不存明文),工作階段用 HttpOnly cookie 維持,並對登入嘗試做節流。務必搭配 HTTPS(由前面的反向代理 / 平台終結 TLS),cookie 才會帶上 `Secure`。

### 一鍵部署到 VPS(docker compose + 自動 HTTPS)

專案內含 `docker-compose.yml` 與 `Caddyfile`,用 [Caddy](https://caddyserver.com/) 反向代理自動申請 / 續期 Let's Encrypt 憑證,讓你在 VPS 上幾個步驟就上線:

```bash
# 0) 先把網域的 A 記錄指向這台 VPS 的 IP(例:nuwa.example.com → 203.0.113.10)
# 1) VPS 上安裝 Docker(含 compose 外掛),然後取得程式碼
git clone https://github.com/chung223/mindkiln.git && cd mindkiln

# 2) 建立 .env,填入網域、登入密碼與(建議)API 金鑰
cp .env.example .env
nano .env      # 至少填 DOMAIN 與 NUWA_PASSWORD;ANTHROPIC_API_KEY 建議也填

# 3) 啟動(首次會建置映像;Caddy 會自動處理 HTTPS 憑證)
docker compose up -d --build

# 開 https://你的網域 —— 進站需輸入 NUWA_PASSWORD
```

只有 Caddy 對外開放 80/443,女媧工坊本身只在內部網路,外界一律經 HTTPS 進來。資料存於 `nuwa-data` volume,重建容器不遺失。更新版本:`git pull && docker compose up -d --build`。

> 未設定 `NUWA_PASSWORD` 時 compose 會直接報錯拒絕啟動,避免不小心把無密碼的站開上公網。

### 安裝成 App(PWA)

本站是 PWA:用手機或桌面 Chrome / Edge 開啟後,可從瀏覽器選單「安裝 / 加入主畫面」,像原生 App 一樣獨立視窗開啟,離線也能開啟外殼(需連線才能蒸餾 / 對話)。iOS Safari 用「分享 → 加入主畫面」。

## 使用流程

1. **建立人物** — 點「造一個新人物」,輸入名稱與補充說明。若語料是**多人對話**(群組聊天、訪談逐字稿),可填「對話中的稱呼/帳號」——蒸餾時只鎖定此人的發言,別人對他的評價則歸入「他者視角」維度(此欄位建立後也可在人物頁隨時修改)。
2. **放入語料** — 把此人的相關文件丟進 `data/characters/<人物>/sources/`(介面上可一鍵開啟資料夾),或直接拖放上傳。支援格式:`txt / md / pdf / docx / srt / vtt / html / json / csv`。一手素材(本人著作、訪談逐字稿、字幕)品質遠勝二手轉述。
3. **入爐蒸餾** — 選擇檔位後開始:
   - **標準檔**:六維度完整分析(著作、對話、表達DNA、他者視角、決策、時間線)
   - **快速檔**:三維度(著作、對話、表達),省時省費
   蒸餾管線:維度分析 → 心智模型三重驗證提煉(跨域復現/生成力/排他性)→ 組裝 `persona.md`。調研中間產物存於 `research/`,全程可在介面觀看進度。
4. **對話 · 預測 · 感情處理**（開新對話時選模式)
   - **對話模式**:以此人身份角色扮演,遵循表達DNA與誠實邊界
   - **預測模式**:描述一個情境,預測此人會如何反應/決策,附推理依據、信心度與反向情境
   - **情境條件**:開新對話時可設定情境、時間點、你的身份、表達約束等,整段對話都會遵守
   - **感情處理模式**(借用諮商技術,幫你把一段關係想清楚、往前走):
     - **排練難開口的對話** — 練習說出口不易的話,由對方用可能的方式真實回應
     - **未寄出的信** — 寫下想說卻沒說的,收到一封對方語氣的回信(放下練習)
     - **從對方的角度看** — 讓對方說出當時他那一邊可能的心境,幫你看見全貌
     - **反思陪伴** — 對方陪你聊,同時是一面鏡子,溫柔照見你自己的模式

5. **議事會 Advisory Board**(側欄下方)— 召集 **2 位以上**已蒸餾的人物同場討論同一個問題。他們依序回應,**後發言者看得到前者說了什麼**,可以附和或反駁——像一個真的思維顧問團在你面前議事。

> 誠實提醒:蒸餾出來的是一面鏡子,不是本人。它能幫你把話、把感受、把一段關係看得更清楚,但別把它當成真人的替代品——尤其涉及在世的私人對象時。

## 進階功能

- **人物檔案可編輯**:蒸餾出的 `persona.md` 可在介面直接檢視 / 編輯(改完即刻生效),不必到檔案系統翻找。
- **調研文件檢視**:各維度分析與綜合報告都能在介面打開,審視「為什麼提煉出某個心智模型」;每個維度可**單獨重跑**再自動重建人物檔案,不必整條重來。
- **品質稽核閘**:綜合提煉後,獨立的稽核步驟對照評分表打分(心智模型數量、內在張力、引語可溯源、排他性),不合格會帶著具體批評自動重跑(最多 2 輪)。強模型(Claude / MiniMax)效果最好;小模型無法產出結構化評分時會優雅略過、不阻擋交付。
- **多人對話**:填此人在紀錄裡的稱呼即可鎖定;或按「偵測發言者」自動掃出候選讓你勾選。WeChat / LINE / WhatsApp 匯出的時間戳、系統訊息、貼圖佔位符會自動清洗。
- **對象類型與同意**:建立人物時標記公眾 / 私人 / 自己 / 虛構;私人對象需勾選同意確認(蒸餾在世非公眾人物涉及隱私)。
- **輸出語言**:預設繁中,也可選簡中 / 英文 / 日文 / 跟隨語料——蒸餾英日語料時不會被強制轉成中文。
- **繁體變體**:「台灣正體 + 詞彙」或「只轉字不改詞」(蒸餾大陸講者時保留他真實用語)。
- **成本試算**:蒸餾前按「試算」估算字數、呼叫次數與 token,再決定要不要開跑。
- **分模型省錢**:維度分析(機械式抽取)可指定便宜模型,綜合與 persona 組裝仍用主模型。
- **語料快取**:同一份語料在多個維度間以快取前綴重用,雲端 / MiniMax 大幅省 input 費用。
- **一鍵備份**:設定頁「備份資料」把整個 `data/` 打包下載。

## 資料結構

```
data/characters/<人物>/
├── character.json   # 人物中繼資料
├── sources/         # ← 把文件丟進這裡
├── research/        # 蒸餾中間產物(各維度分析 + 綜合報告)
├── persona.md       # 蒸餾成果:可運行的人物思維檔案
└── chats/           # 對話紀錄
```

所有資料都在本機,`persona.md` 可自由編輯(改完即刻生效)。

## 設定

左下角「設定」可選**模型來源**:

- **Anthropic 雲端**(預設,品質最佳):填入 API 金鑰,模型可選 `claude-opus-4-8` / `claude-sonnet-5` / `claude-haiku-4-5`。
- **Anthropic 相容端點 · MiniMax 等**:走 Anthropic Messages API 相容端點,重用雲端串流邏輯。填 Base URL(MiniMax 為 `https://api.minimaxi.com/anthropic`)+ 模型名稱(`MiniMax-M3` 有 1M 上下文,蒸餾建議;或 `MiniMax-M2.7` 等)+ API 金鑰。MiniMax 中文品質好、有你的 token plan,但預設吐簡體——**建議搭配下方「強制繁體輸出」**。
- **本地模型 · OpenAI 相容**:接 Ollama、LM Studio、llama.cpp、vLLM 等本地推論伺服器(填 Base URL 如 `http://localhost:11434/v1` + 模型名稱,金鑰選填)。語料完全不出本機,適合蒸餾私人素材。
  - ⚠ 本地小模型的**蒸餾**品質(尤其心智模型提煉)明顯不如雲端 Opus,建議搭配快速檔或大參數量的長上下文模型;**對話**模式則較友善。切到本地時語料上限自動下修為 12 萬字。

**強制繁體輸出(OpenCC)**(預設開啟):蒸餾產出與對話回覆一律經 OpenCC `s2twp` 轉為繁體中文(台灣用語,連詞彙都轉:软件→軟體、信息→資訊、鼠标→滑鼠)。確定性、離線、零成本,對已是繁體的輸出(如 Claude)無害。非 Claude 模型(MiniMax、本地模型常吐簡體)務必保持開啟。

其他:

- **語料上限**:雲端約 50 萬字、本地約 12 萬字,超出時各文件按比例截斷(介面會提示)。
- **費用提醒**:雲端蒸餾是多次長文本模型呼叫,語料越多費用越高。先用快速檔試效果,再跑標準檔。本地模型則免費。

## 誠實邊界

蒸餾產出的不是本人,是從公開/提供素材中提煉的思維框架鏡像。它不能預測本人對全新問題的真實反應,公開表達與真實想法可能有落差。蒸餾在世的非公眾人物前,請先徵得當事人同意。
